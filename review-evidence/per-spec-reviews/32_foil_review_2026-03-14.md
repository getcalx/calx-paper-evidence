## Blocking Issues (Nolan)

### B1: Wrong file path in 32J
`services/agent.py` does not exist. The handler lives at `agent/tools/analysis.py`. Fix file references in 32J chunk table.

### B2: `confidence_alpha` and `confidence_beta` typed as INT — must be FLOAT
Beta distribution parameters are real-valued. INT truncates every fractional Bayesian update to zero. Fix in both migration DDL and Pydantic model.

### B3: Two competing confidence update mechanisms
Spec declares Beta params (`confidence_alpha`, `confidence_beta`) but the `bayesian_confidence_update` function uses direct scalar math on the `confidence` float and never reads/writes alpha/beta. Pick one:
- **Option A:** Drop alpha/beta. Use the scalar approach (simpler, already defined).
- **Option B:** Use real Beta distribution: `confidence = alpha / (alpha + beta)`. Accept increments alpha, reject increments beta.

**Bryce decides.** Either works. Having both is a contradiction.

### B4: YAML boolean coercion not closed in Pydantic models
`status: yes` in YAML silently becomes `status = "True"` in Python. Default Pydantic coerces `True` to `"True"` in `str` fields. Add `@field_validator` on `ObsidianRoleNote.status`, `ObsidianClusterNote.cluster_label`, and any string field where YAML boolean coercion could produce a valid-but-wrong value. Use `StrictStr` or explicit bool rejection.

### B5: `translation_context` NOT NULL but missing from `CatalogCorrectionCreate`
DB declares `translation_context JSONB NOT NULL`. Request model has no `translation_context` field. Clarify: who calls `POST /v1/catalog/corrections`? If internal-only (derived from card_interactions), remove the public endpoint. If frontend calls it, add the field.

### B6: `role_alias_misses` RLS blocks anonymous users
INSERT policy requires `authenticated` role. Anonymous users use `anon` key. Alias miss logging should work for anonymous users (it's just logging, not corrections). Add anon INSERT policy.

### B7: `competency_clusters` missing `created_at`
Other tables have it. Add `created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`. Free now, migration later.

### B8: `onet_soc_code` singular/plural mismatch
Obsidian frontmatter has `onet_soc_code` (singular TEXT). DB has `onet_soc_codes` (plural JSONB array). `parse_role_note` reads singular. Fix: change Obsidian schema to plural with JSONB format, or add parser transformation.

## Non-Blocking Flags (Nolan)

### F1: Fan-out ordering — one bad reference kills entire sync
Transactional sync means one unresolvable cluster→role reference rolls back everything. Consider logging + skipping unresolvable membership links instead of failing the whole transaction.

### F2: File rename + title change = orphan + new row
Acceptable for single-author vault. Document the constraint.

### F3: spaCy model download not in Dockerfile
`python -m spacy download en_core_web_sm` must be in the Dockerfile or CI. pip install alone won't do it.

### F4: FE Contract Updates section missing
New API response types need documented TS contract updates per Bryce's Rule 26.

## Thinker Recommendations

### Reviewer-BH6 — Affective signal is underweighted
- **Correction weights inverted:** Affective rejections (>3s dwell + hovers) weighted 0.6x when they contain the most differentiating signal. Should be highest weight, not lowest.
- **Register should be first-class dimension**, not nullable tag. Functional/narrative/adversarial register changes the translation operation.
- **MCP conversation capture (Wave 7+) should ship sooner** — Spencer's authoring sessions are ground-truth emotional intelligence about vocabulary register. Defer = ship without the moat.
- **Identity rejections (revert after edit)** should trigger clarifying questions, not just confidence decay.

### Reviewer-SY4 — Representation and sampling concerns
- **Confidence initialization 1.0 contradicts Beta(1,1) prior.** Uniform prior = 0.5, not 1.0. Bootstrap phrases start maximally confident, system is slow to surface bad translations.
- **Tech-company-only corpus bias.** 11 roles all source from ~10 Series B+ tech companies. GTM/Sales/HR vocabulary at Stripe ≠ healthcare/finance/manufacturing. `culture_signal` tag has zero enterprise data.
- **100 JDs may be too small for HDBSCAN** at `min_cluster_size=5`. Consider expert-defined initial clusters with embedding assignment.
- **Correction data IS training data.** The 5000-correction supervised threshold should be near-term priority, not Phase 2 afterthought.
- **Cross-role vocabulary patterns** (phrases in 5+ catalogs) should start with higher confidence — validated across contexts.
