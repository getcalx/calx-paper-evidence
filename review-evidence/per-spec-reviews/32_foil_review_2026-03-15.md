## Round 2 Blocking Issues

### B1: `scalar_confidence_update` cannot receive affect proxy signals (Reviewer-BH6)

The function signature takes `interaction_type` and `user_tier` but NOT `dwell_time_ms`, `hover_count`, or `reverted`. The correction profile classification (semantic/affective/identity at lines 1708-1711) is documented but the function that performs the update has no parameters to receive that signal. Broken wire.

**Fix:** Add `correction_profile: Literal["semantic", "affective", "identity"]` parameter to `scalar_confidence_update`, or have `record_correction` classify the profile first and pass profile-adjusted weight.

### B2: Correction weight composition unspecified (Reviewer-BH6)

Three weight dimensions exist: tier (1.0/0.6/0.3/0.5), profile (1.0/0.6/1.2), source (1.5 for MCP). The formula uses one `weight` variable. How do they compose?

**Fix:** Specify the composition rule. Recommend multiplicative with floor: `final_weight = max(tier_weight * profile_multiplier * source_weight, 0.1)`.

### B3: `insert_vocabulary` missing `register` and `source` parameters (Reviewer-BH6)

Columns exist on `cluster_vocabulary`. The insertion function cannot set them. Signature gap.

**Fix:** Add `register` and `source` to `insert_vocabulary` parameter list.

### B4: Affective rejection weight still inverted at 0.6x (Reviewer-BH6)

A deliberate, dwelled-upon rejection (>3s dwell + hovers) is weighted LOWER than a snap rejection (1.0x). Reviewer-BH6 argues this should be at least 1.0x — affective rejections contain the most differentiating signal about voice/register misalignment.

**Fix:** Change affective rejection weight from 0.6x to at least 1.0x. Update acceptance criteria at line 1734.

### B5: GapCard missing `catalog_cluster_id` and `catalog_cluster_label` (Sloane)

FE Contract section says GapCard gets these fields. Implementation instructions only add them to TranslationCard. Gap cards will fall out of cluster groups in the UI.

**Fix:** Add the two optional fields to `GapCard` in 32B Pydantic models. Add population instructions in 32J.

### B6: Five public catalog TS types undefined (Sloane)

`CompetencyCatalog`, `CompetencyCluster`, `ClusterVocabularyEntry`, `DiscriminativeMarker`, `LevelSignal` have Pydantic models but no TypeScript definitions in the FE Contract section. Faye cannot write interfaces from one-line descriptions.

**Fix:** Add complete TS interface definitions for all 5 types, field-for-field.

### B7: `AliasCreated` type referenced but never defined (Sloane)

FE Contract uses `AliasCreated[]` — type doesn't exist anywhere in the spec. Pydantic model uses `list[dict]`.

**Fix:** Define `AliasCreated` as a Pydantic model with TS definition, or inline the shape.

### B8: Missing `catalog_cluster_sort_order` on card payload (Vivian)

Faye has no way to render cluster group headers in correct order. `catalog_cluster_membership.sort_order` never makes it into the SSE card event.

**Fix:** Add `catalog_cluster_sort_order?: number` to `TranslationCardData` and `GapCard`. Denormalize alongside `catalog_cluster_label`.

### B9: No guaranteed emission ordering by cluster in 32J (Vivian)

Cards emit in LLM output order, not cluster order. Without ordering, cluster group headers appear randomly during the streaming reveal — visual chaos.

**Fix:** The assembler in 32J must sort cards by `catalog_cluster_sort_order` before streaming via SSE. Cards within the same cluster must be consecutive. Add to 32J acceptance criteria.

### B10: Add `min_similarity` threshold to `match_vocabulary` (Reviewer-SY4)

Without a floor, the analogizer returns the nearest neighbor regardless of distance. A resume phrase about P&L management could match a weak catalog phrase about sprint velocity.

**Fix:** Add `min_similarity: float = 0.65` parameter and filter results below it.

### B11: Add TTFT latency budget for catalog-grounded translations (Reviewer-DS2)

The spec does not state expected time-to-first-token delta from catalog prompt injection (~3,600 extra tokens in system prompt → ~1-2s TTFT increase). No token ceiling for the catalog grounding block.

**Fix:** Add `CATALOG_PROMPT_TOKEN_BUDGET` config (default 4,000). Truncate/prioritize by confidence if exceeded. State expected TTFT delta.

### B12: Confirm SSE connection topology (Reviewer-DS2)

Direct (frontend → Railway) vs proxied (frontend → Vercel API route → Railway). Timeout implications differ (60s Vercel vs 300s Railway).

**Fix:** Document the SSE connection topology and timeout implications.

### B13: Fix "0ms incremental cost" claim (Reviewer-DS2)

Phase 2 JD embedding comparison is <5ms, not 0ms. Accurate claims build trust.

**Fix:** Change to "<5ms incremental cost, piggybacking on already-computed embedding."

## Non-Blocking Issues

### NB1: Document tech-company corpus bias (Reviewer-SY4)

All 11 bootstrap source boards are VC-backed tech companies. `culture_signal` tag exists but data is homogeneous. Add a "Known Limitations" entry.

### NB2: Cross-role vocabulary patterns not implemented (Reviewer-SY4)

Phrases appearing in 5+ catalogs should get confidence boost. Data model supports the query but logic doesn't exist. Track for Phase 2.

### NB3: Missing imports in models code block (Nolan)

`Any` and `field_validator` used but not imported. Bryce will catch at build time.

### NB4: `record_correction` function signature gap (Nolan)

Acceptance criteria say it takes `translation_context` but the function stub doesn't show it.

### NB5: Add `affective_reject_count` counter to `cluster_vocabulary` (Reviewer-BH6)

Cheap INT counter. Preserves affect signal dimension for Phase 2 supervised learning without schema redesign.

### NB6: Add test cases for correction profiles (Reviewer-BH6)

No tests for semantic/affective/identity detection or profile-tier weight interaction.

### NB7: Annotate vocabulary with `register` in prompt template (Reviewer-BH6)

Even without register-based filtering, annotating phrases in the LLM prompt helps the model distinguish functional vs narrative register.

### NB8: Error codes not formalized for catalog endpoints (Sloane)

404 and 409 mentioned but no `error_code` values specified for frontend conditional rendering.

### NB9: `docs/frontend-types.ts` update missing from 32F acceptance criteria (Sloane)

The spec says to update it but the AC doesn't enforce it.

### NB10: Consider aggressive phasing (Reviewer-DS2)

Ship read path (schema + models + catalog lookup + prompt injection + basic Obsidian sync + alias lookup) as Phase 1. Defer bootstrap pipeline (32E, 32G), corrections system (32D), HDBSCAN/UMAP/spaCy to Phase 2. Cuts build from ~3 sessions to ~1.5.
