# Foil Review: Spec 34 — Tier 2 Experience Path

**Date:** 2026-03-15
**Reviewers:** Nolan (backend foil), Sloane (FE contract + Reviewer-BH6/Reviewer-DS2/Reviewer-SY4), Vivian (design + Reviewer-BH6/Reviewer-DS2/Reviewer-SY4)
**Overall verdict:** REVISE — 19 blocking issues across 3 reviewers, 0 architectural changes required

---

## Blocking Issues

### B1: `signal_type` column does not exist on `level_signals` (Nolan + Sloane)

The query SELECTs `ls.signal_type` from `level_signals`. Migration 023 defines exactly four columns: `id`, `cluster_id`, `seniority_level`, `signal_phrase`. No `signal_type`. Column verification table falsely marks it verified. Query will crash at runtime.

**Fix:** Remove `ls.signal_type` from the SELECT. Nothing in the logic uses it after selection.

### B2: `competency_clusters` has no `catalog_id` — join path is wrong (Nolan + Sloane)

The query uses `WHERE cc.catalog_id = $1`. Spec 32 explicitly says: "Note: `catalog_id` FK removed from this table. Use `catalog_cluster_membership` instead."

**Fix:** Rewrite all queries to join through `catalog_cluster_membership`:
```sql
JOIN catalog_cluster_membership ccm ON ccm.cluster_id = cc.id
WHERE ccm.catalog_id = $1
```

### B3: `_analyze_vocabulary` does not accept `resume_embedding` or `override_catalog_id` (Nolan)

The spec passes `resume_embedding=resume_embedding` and `override_catalog_id=better_fit_catalog_id` to `_analyze_vocabulary`. The current signature is `(pool, user_id, input)` — neither parameter exists.

**Fix:** Document exact signature changes to `_analyze_vocabulary` and how new params flow through to `classify_tier`.

### B4: Better-fit analysis uses the aspirational JD, not the matched better-fit JD (Nolan + Sloane)

At line 475, better-fit analysis reuses `jd_text=jd_text` (the original aspirational JD). This produces gaps against the aspirational target with a different catalog — not "where your experience already maps." The whole point is to analyze against a JD at the user's level.

**Fix:** `_find_better_fit_role` should return the matched `bootstrap_jds` row's `jd_text`. Use that for the better-fit analysis.

### B5: `role_title_raw` vs canonical `role_title` mismatch (Nolan + Sloane)

`_find_better_fit_role` returns `role_title_raw` (e.g., "Senior Product Manager, Growth"). `get_better_fit_catalog` queries `WHERE role_title = $1` against canonical titles (e.g., "Product Manager"). Will silently fail → no better-fit cards ever shown.

**Fix:** Use the `catalog_id` FK from the matched `bootstrap_jds` row directly, rather than going through a title-based lookup.

### B6: No `aspirational` TierSectionEvent emitted (Nolan + Sloane)

The model defines `section: Literal["aspirational", "better_fit"]` but code only emits `section="better_fit"`. Frontend has no machine-readable aspirational role title for the section header.

**Fix:** Emit `TierSectionEvent(section="aspirational", ...)` before the first aspirational card. Stream becomes: `session_init` → `tier` → `tier_section(aspirational)` → cards → `tier_section(better_fit)` → cards → `done`.

### B7: Embedding thresholds declared but never used in classification (Nolan + Sloane + Reviewer-SY4)

`TIER2_ASPIRATIONAL_SIM_CEILING` (0.40) and `TIER2_LEVEL_MISMATCH_THRESHOLD` (3) are declared but never referenced. Key Decision 1 promises embedding similarity classification. Actual implementation is pure substring matching. The `resume_embedding` parameter is only used for better-fit search, not classification.

**Fix:** Either (A) implement the embedding similarity check from Key Decision 1 — compare `resume_embedding` against cluster embeddings at different seniority levels, or (B) remove the constants and update Key Decision 1 to honestly describe the vocabulary-match-count algorithm. Option A recommended.

### B8: Missing Spencer Config section (Nolan)

Bryce's rules require all configurable values in a Spencer Config section. Three thresholds + Voyage API cost increase (1 embed call per resume) are undocumented.

**Fix:** Add Spencer Config section: `TIER2_ASPIRATIONAL_SIM_CEILING`, `TIER2_BETTER_FIT_SIM_FLOOR`, `TIER2_LEVEL_MISMATCH_THRESHOLD`, and cost impact note.

### B9: `_build_tier2_reason` copy violates PRD framing constraints (Reviewer-BH6 via Sloane)

Copy: "Your experience pattern is strongest at the individual contributor level. This role targets director-level vocabulary."

PRD §4 constraint #1: "Lead with what's true about them. Mirror competencies with high specificity before any redirect." Copy leads with level assessment, zero competency specificity.

PRD §4 constraint #3: "Show recognition before title. Competency mapping first, role title second." Copy shows levels first.

**Fix:** Reason builder needs access to strongest cluster match data. Generate copy like: "Your [stakeholder alignment] vocabulary is sharp. This role targets director-level framing. Here's where the language shifts." Consider structured `tier2_context` field on `TierEvent` instead of prose string — let FE compose display copy.

### B10: No performance budget for dual-LLM stream (Reviewer-DS2 via Sloane + Vivian)

Spec claims "~110ms overhead." Ignores the second full LLM call for better-fit cards (2-5s). Actual Tier 2 timeline: ~4-10 seconds sequential. User stares at `tier_section` separator with no cards for 2-5s after aspirational cards complete.

**Fix:**
1. Add targets: first aspirational card < 3s, first better-fit card < 2s after tier_section
2. Parallelize: start better-fit analysis concurrently with aspirational (both inputs known after classification). Buffer better-fit cards, emit after separator.
3. If not parallelizable, add loading state contract to `tier_section` event.

### B11: `tier_section` separator has no design spec (Vivian)

The event is the emotional pivot point of the entire Tier 2 experience. No visual treatment defined. Design spec Section 9 describes a header + agent bubble, but the `tier_section` event implies a single unified surface. These conflict.

**Fix:** Spec 34 must state whether `tier_section` renders as (A) standard agent bubble + header label, or (B) new compound component. If B, needs visual spec before build.

### B12: No emotional breathing room at separator (Reviewer-BH6 via Vivian)

Design spec prescribes 300ms stagger (double normal). Not enough. User just absorbed gap cards — emotional state is vulnerability. 300ms is perceptually indistinguishable from normal flow. The reframe arrives before the gap has settled.

**Fix:** Minimum 1200-1600ms hold after tier_section before first better-fit card (matches "That's the gap" hold timing from design spec Section 8).

### B13: Time-to-first-token for better-fit group unspecified (Reviewer-DS2 via Vivian)

Two TTFT moments in Tier 2. The second (after separator) depends on LLM processing for the second analysis. A stall after "here's where you shine" with nothing below it is devastating.

**Fix:** Set target: < 1500ms from tier_section to first better-fit card. If backend can't guarantee, kick off better-fit analysis in parallel with aspirational cards.

### B14: Graceful degradation produces broken emotional arc (Reviewer-SY4 via Vivian)

When no better-fit catalog exists: tier event says tier=2, aspirational gap cards stream, no reframe arrives. User gets only the vulnerability half. The system told itself this person is overreaching, showed gaps, offered nothing.

**Fix:** If no better-fit catalog exists, do NOT classify as Tier 2. Fall through to Tier 3. Classification should depend on catalog availability, not just signal matching.

## Non-Blocking Issues

### NB1: Substring matching is fragile (Reviewer-SY4 via Nolan + Sloane)

`if phrase in resume_lower` misses word order/form variations. "Oversaw budget allocation" won't match "budget oversight." Consider token-overlap scoring (Jaccard on word sets) as intermediate between exact substring and full embedding.

### NB2: Two `pool.acquire()` calls in `_find_better_fit_role` where one suffices (Nolan)

Sequential queries should share a connection to reduce pool pressure.

### NB3: Copy is placeholder subject to brand review (Nolan)

PRD §4 has five non-negotiable design constraints. Final copy should be validated against all five by Morgan/Sloane.

### NB4: No `docs/frontend-types.ts` update in acceptance criteria (Nolan + Sloane)

Every new SSE event type needs it. Add to AC and chunk plan.

### NB5: Missing import for `TierSectionEvent` in catalog.py code block (Nolan)

### NB6: No test for extreme downward mismatch (VP resume + IC JD) (Nolan)

Edge case worth covering for seniority order boundary correctness.

### NB7: All `classify_tier` callers must be updated from sync to async (Nolan + Sloane)

Spec changes `def` to `async def`. Every existing caller needs `await`. List ALL call sites, not just the new signature.

### NB8: Tied level counts resolve silently (Sloane)

`max()` on dict returns first encountered on tie. May bias toward lower seniority (more Tier 2 false positives). Document if intentional per Key Decision 8.

### NB9: Missing error/empty states for `tier_section` (Sloane)

What if better-fit analysis returns zero cards or errors mid-stream? Separator with nothing below it is worse than no separator. Add `tier_section_done` or FE timeout behavior.

### NB10: No classification quality logging (Reviewer-SY4 via Sloane)

No instrumentation for evaluating accuracy. Log: `session_id`, `user_level`, `jd_level`, `level_match_counts`, `tier_result`, `better_fit_found`, `better_fit_similarity`.

### NB11: Reattribution line ownership still unresolved (Reviewer-BH6 via Sloane)

"You described [X] skills that are in demand. You just used different words." Matters most in Tier 2. Which spec owns it?

### NB12: 150ms stagger instead of design spec's 300ms after separator (Sloane)

Design spec says 300ms between last aspirational card and first better-fit card. Spec uses 150ms.

### NB13: `seniority_tag` is nullable but used as non-null filter (Sloane)

`WHERE seniority_tag = $3` excludes all untagged JDs. Add `OR seniority_tag IS NULL` or document requirement.

### NB14: Seniority labels need affective calibration (Reviewer-BH6 via Vivian)

"Individual contributor" reads as demotion label. Consider scope-based framing: "team-scope" vs "organization-scope." Spencer's call on copy.

### NB15: Inter-card stagger compression for long streams (Reviewer-DS2 via Vivian)

12-16 cards at 150ms + 800ms reveal = 10-12s total. After first 3-4 cards establish pattern, compress to 100ms.

### NB16: Classification confidence not surfaced to FE (Reviewer-SY4 via Vivian)

No confidence score on tier event. Marginal classifications get same treatment as confident ones. Consider `confidence: "high" | "moderate"` for future.

### NB17: Level pill integration with tier_section role title (Vivian)

Design spec's Reviewer-LP3o Ash level pill is only specified for inline card text, not section headers. Extend to tier_section display.

### NB18: Tier 2 summary share action (Vivian)

Per-card share exists but no section-level share. Composite image with both role titles + pivot copy = the LinkedIn share artifact. Potentially the most screenshottable moment in the product.
